Project Overview

This repository contains Part 2 of the End-to-End Applied AI & ML Data Product Capstone Project.

The objective of this project is to build a production-ready Machine Learning pipeline that predicts cybersecurity incident risk using historical incident data. The project transforms cleaned cybersecurity data into predictive models capable of estimating future incident severity or risk levels.

The implementation follows modern machine learning engineering practices including data preprocessing, feature engineering, feature selection, model training, model evaluation, hyperparameter tuning, model persistence, and reproducible experimentation.

This repository answers the second analytics question:

«Predictive Analytics – What is likely to happen?»

The trained model produced in this repository will be used by the interactive dashboard (Part 3) and AI-powered assistant (Part 4).

---

Business Problem

Cybersecurity teams receive thousands of security alerts every day. While historical reports explain what has already happened, organizations also need to anticipate future risks.

Manual prioritization is slow and inconsistent, making it difficult to identify high-risk incidents early.

This project develops a Machine Learning solution capable of learning patterns from historical cybersecurity incidents and predicting future incident severity or risk, helping analysts focus on the most critical events first.

---

Analytics Objective

This repository focuses on Predictive Analytics.

The project aims to answer the following business questions:

- Which cybersecurity incidents are likely to become high risk?
- Can incident severity be predicted from historical patterns?
- Which features contribute most to cybersecurity risk?
- Which Machine Learning algorithm performs best?
- How accurately can future incidents be classified?

Dataset

This project uses the cleaned and feature-engineered cybersecurity incident dataset prepared during Part 1.

The dataset contains information such as:

- Incident ID
- Incident Date
- Attack Type
- Severity Score
- Industry Sector
- Region
- Threat Actor
- Records Affected
- Downtime Hours
- Financial Impact
- Regulatory Fine
- Response Team Size
- Engineered Features

Dataset location:

data/processed/engineered_incidents.csv

If the processed dataset is unavailable, it can be regenerated by running the Part 1 ETL pipeline.

Machine Learning Objective

The pipeline performs the following stages:

- Data Loading
- Data Validation
- Feature Encoding
- Missing Value Handling
- Feature Scaling (where required)
- Feature Selection
- Train/Test Split
- Model Training
- Model Evaluation
- Model Comparison
- Best Model Selection
- Model Saving

The final trained model is stored for future inference and dashboard integration.
