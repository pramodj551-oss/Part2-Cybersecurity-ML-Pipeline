Versioning Strategy

This project follows Semantic Versioning.

- Major Version – Breaking architectural or functional changes.
- Minor Version – New features without breaking compatibility.
- Patch Version – Bug fixes, documentation updates, and performance improvements.

Example:

- "1.0.0" → Initial Release
- "1.1.0" → New Feature
- "1.1.1" → Bug Fix
- "2.0.0" → Major Machine Learning Release
